# 万万烧烤小程序验收清单

## 前端页面

- 顾客端：选桌、扫码绑定桌号、菜单、规格弹层、加购/减购、购物车抽屉、确认下单、成功页。
- 店员端：登录、订单工作台、新订单/待完成/已完成筛选、确认/完成订单、桌号聚合。
- 管理端：菜单管理、分类管理、菜品编辑、门店管理、头图上传、桌码生成。

## 组件规范

- `DishRow`：商品图片、信息、操作三列稳定；多规格菜品行内减购要扣减已有规格项。
- `ProductSummaryRow`：弹层和购物车商品摘要复用同一组件。
- `QuantityStepper`：视觉按钮 48rpx，点击热区足够，不能被 footer 遮挡。
- `BottomSheet`、`CartBar`：固定底部区域必须保留安全区。
- `EmptyState`、`ErrorState`：加载失败和空数据页面必须有恢复入口。

## CloudBase 接入

1. 在 `miniprogram/utils/config.js` 中填写真实 `ENV_ID`。
2. 在 `cloudbaserc.json` 中填写同一个真实 `envId` 和一次性 `SETUP_KEY`。
3. 部署 `cloudfunctions/wanwanApi`。
4. 调用 `initSeedData` 初始化：`settings`、`tables`、`categories`、`dishes`、`staff_users`、`operation_logs`。
5. 当前 `USE_MOCK = false`，没有真实环境时会调用云函数失败。
6. 真实试营业前必须替换默认店员账号和密码。

## 必测场景

- 扫码或手动选桌进入菜单，桌号正确。
- 普通菜直接加购；规格菜选择规格后加入购物车。
- 菜单行、规格弹层、购物车抽屉数量保持同步。
- 收起、返回、重新进入菜单后购物车仍能恢复。
- 提交订单时按钮进入 loading，重复点击不创建多单。
- 提交失败时保留购物车并显示错误。
- 店员端新订单出现后可确认、完成，并可在已完成筛选中查看历史。
- 管理端保存门店头图、菜品、分类后，顾客端刷新生效。
- 门店管理生成桌码后，微信扫码能进入对应桌号菜单。
- 弱网/接口失败时显示错误态或 toast，不出现空白页。
